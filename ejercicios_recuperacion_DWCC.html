Semanas 1-2: Git · Herramientas · JavaScript Sintaxis

23 feb – 6 mar
Git

1.1 Flujo básico de Git Fácil

Crea un repositorio local llamado practica-git. Realiza las siguientes operaciones y anota los comandos usados:

  • Inicializa el repositorio con git init.
  • Crea un archivo index.html con una estructura HTML básica.
  • Añade el archivo al staging area y haz un commit con el mensaje "Primer commit".
  • Crea un archivo .gitignore que ignore archivos .log y la carpeta node_modules/.
  • Conecta el repositorio local con un repositorio remoto en GitHub y sube los cambios.

1.2 Ramas y fusión Media

Partiendo de un repositorio con al menos un commit en la rama main:

  • Crea una rama llamada feature-header y cámbiate a ella.
  • Modifica index.html añadiendo un <header>. Haz commit.
  • Vuelve a main y modifica index.html añadiendo un <footer> (en zona distinta). Haz commit.
  • Fusiona feature-header en main. ¿Se produce fast-forward o fusión automática? ¿Por qué?
  • Elimina la rama feature-header.
  • Visualiza el historial con git log --oneline --graph --decorate --all y explica el resultado.

1.3 Resolución de conflictos Media

Provoca un conflicto de fusión intencionadamente:

  • Crea una rama conflict-branch. Modifica la línea 5 de un archivo. Haz commit.
  • Vuelve a main. Modifica la misma línea 5 del mismo archivo con contenido distinto. Haz commit.
  • Intenta fusionar. Resuelve el conflicto manualmente en el editor.
  • Completa la fusión con git add y git commit.

1.4 Stash y etiquetas Fácil

  • Modifica un archivo sin hacer commit. Guarda los cambios con git stash.
  • Comprueba que el directorio de trabajo está limpio con git status.
  • Haz otro cambio, haz commit, y después recupera el stash con git stash pop.
  • Crea una etiqueta anotada v1.0 con el mensaje "Primera versión estable".
